\begin{abstract}
This paper studies state uplift across nested operating regimes in Gemma 3n. Gemma 3n is distributed as separate E2B and E4B artifacts, but its MatFormer-style design makes the reduced E2B-like regime part of the same parent family rather than an unrelated checkpoint. That architectural fact motivates a narrower and more testable question than heterogeneous model bridging: can internal state from the reduced operating regime be corrected just enough for decoding to resume under the fuller regime with improved behavioral fidelity? We investigate this question using paired low/full runs on identical prompt prefixes, residual-state translators with null controls, live handoff metrics at horizons 1/4/8/16, and a sequence of kill-tests that isolate where intervention actually matters. The main empirical result is that broad residual training is insufficient, while a targeted residual uplift at \method{layer34,last1} produces strong continuation recovery and generalizes to a held-out slice. The resulting picture is clear: the dominant bottleneck is intervention placement and regime-specific control, not generic state-fitting capacity.
\end{abstract}

\section{Introduction}
The project in this repository began with a broader bridge hypothesis: can one transplant reasoning state across distinct language models? That hypothesis was too unconstrained. RegimeLift narrows the problem to a setting where structural relatedness is real and measurable: regime handoff inside Gemma 3n's shared E2B/E4B family.

Gemma 3n is documented as a nested / selective-activation architecture in which reduced and fuller operating modes coexist inside one parent family \cite{gemma3n2025,mrl2022}. The E2B and E4B artifacts are published separately, but the scientific object of interest here is not transfer between unrelated checkpoints. It is uplift between two operating regimes that were trained as part of the same nested family. That difference matters. If the reduced regime is an embedded sibling of the fuller regime, then a useful handoff might require only a small corrective transport map rather than a universal latent bridge.

This paper asks a deliberately limited question:
\begin{quote}
Can a low-activation Gemma 3n prefix state be uplifted into the corresponding fuller-regime state so that decoding resumes closer to native full-regime behavior?
\end{quote}

The answer is now evidence-based rather than speculative. Across completed separation tests, null controls, layer sweeps, suffix-span sweeps, and targeted-site studies, the work supports four concrete claims:
\begin{enumerate}[leftmargin=1.4em]
    \item low/full regimes are genuinely separated;
    \item identity is an exact control for the live handoff path;
    \item local state alignment alone is not enough;
    \item a specific learned intervention site, \method{layer34,last1}, yields the first robust continuation recovery result.
\end{enumerate}

\paragraph{Contributions.}
The current draft makes four contributions.
\begin{itemize}[leftmargin=1.4em]
    \item It formulates same-parent regime uplift in Gemma 3n as a falsifiable handoff problem rather than a generic manifold-alignment claim.
    \item It introduces a paired-state extraction and evaluation pipeline that measures both local alignment and resumed decoding behavior.
    \item It shows empirically that intervention site dominates broad translator capacity: broad MLP training improves local metrics, but targeted training at the correct site recovers behavior.
    \item It freezes a first practical learned handoff result at \method{layer34,last1} while documenting the active larger held-out validation as the next gate.
\end{itemize}

\section{Background and framing}
\subsection{From archived He-LMAS to RegimeLift}
The archived He-LMAS project attempted heterogeneous cross-model transfer. That historical work remains useful as context, but it is not the active research track \cite{helmasLegacy2026}. RegimeLift changes the problem class:
\begin{itemize}[leftmargin=1.4em]
    \item Archived He-LMAS: transfer between different models.
    \item RegimeLift: uplift between nested operating regimes of the same parent family.
\end{itemize}

That reframing is not cosmetic. It replaces an ambitious but weakly constrained hypothesis with a narrower one whose failure modes can be isolated experimentally.

\subsection{Why Gemma 3n is the right testbed}
Gemma 3n is explicitly documented as a nested family with selective activation and Mix-n-Match style operating regimes \cite{gemma3n2025}. The reduced E2B-like path and the fuller E4B path are therefore related in a stronger sense than two arbitrary checkpoints. They are separate published artifacts, but they still belong to one shared structural family. For this paper, the important implication is:
\begin{quote}
the reduced path is different enough to create a real handoff problem, yet related enough that a small corrective map is plausible.
\end{quote}

That is the central scientific bet behind RegimeLift.

\section{Related work and positioning}
RegimeLift sits between two literatures. The first is standard transformer language modeling \cite{vaswani2017attention}, which gives the baseline architecture and the residual-stream object we intervene on. The second is nested-representation or Matryoshka-style modeling \cite{mrl2022}, which makes same-family operating-mode comparisons scientifically plausible in the first place.

What RegimeLift does \emph{not} claim is equally important. This is not a universal latent-alignment paper, not a cross-family checkpoint-bridging claim, and not evidence that one model's reasoning can always be transplanted into another. The archived He-LMAS line explored that broader space \cite{helmasLegacy2026}. The current work is intentionally narrower: one parent Gemma family, paired prefix states, and handoff quality measured by resumed decoding rather than geometric aesthetics.

That narrower framing is a strength. It allows the experiments to distinguish three different possibilities:
\begin{enumerate}[leftmargin=1.4em]
    \item the regimes are too similar to justify the project;
    \item the regimes are separated, but only local fit is learnable;
    \item the regimes are separated and a learned intervention can actually recover behavior.
\end{enumerate}
The completed results place RegimeLift in the third case, but only at a specific intervention site.

\section{Problem formulation}
Let $r \in \{\low,\full\}$ denote the reduced and fuller regimes. For prompt prefix $x_{1:T}$, layer $\ell$, and token position $t$, define the residual-stream state
\[
z^{(r)}_{\ell,t} \in \R^d.
\]
The primary objective is to learn a transport map that predicts the fuller-regime state from the reduced-regime state at a chosen intervention site:
\[
\hat{z}^{(\full)}_{\ell,t} = f_\theta\!\left(z^{(\low)}_{\ell,t}, c\right)
= z^{(\low)}_{\ell,t} + \Delta_\theta\!\left(z^{(\low)}_{\ell,t}, c\right),
\]
where $c$ is optional conditioning metadata such as layer index, token position, and regime descriptor.

The project began with broad layer-conditioned translators, but the later evidence showed that targeting a specific site matters more than fitting a globally smooth map.

\subsection{Training objective}
For residual uplift, the current objective is the weighted combination
\[
\mathcal{L}
= \lambda_{\text{state}} \mathcal{L}_{\text{state}}
+ \lambda_{\text{logit}} \mathcal{L}_{\text{logit}},
\]
with
\[
\mathcal{L}_{\text{state}}
= \left\lVert \hat{z}^{(\full)}_{\ell,t} - z^{(\full)}_{\ell,t} \right\rVert_2^2
+ \lambda_{\cos}\left(1 - \cos\!\left(\hat{z}^{(\full)}_{\ell,t}, z^{(\full)}_{\ell,t}\right)\right),
\]
and
\[
\mathcal{L}_{\text{logit}}
= \KL\!\left(
p^{(\full)}_{t+1} \,\middle\|\, p^{(\text{uplift})}_{t+1}
\right).
\]

The deployment criterion is behavioral rather than geometric: improved continuation after handoff. Let $y^{(\full)}_{t+i}$ denote greedy continuation under the fuller regime and $\hat{y}_{t+i}$ the continuation after patching the reduced regime and resuming under the fuller regime. For horizon $H \in \{1,4,8,16\}$, define
\[
\mathrm{Match}@H
= \frac{1}{NH}\sum_{n=1}^N \sum_{i=1}^{H}
\mathbf{1}\!\left[\hat{y}^{(n)}_{t+i} = y^{(\full,n)}_{t+i}\right].
\]
The project only counts as successful when these continuation metrics improve over the proper no-patch baseline.

\section{Experimental design}
\subsection{Paired-state extraction}
For each prompt prefix, the same parent Gemma 3n family is run under two operating modes: a reduced E2B-like regime and a fuller E4B regime. The extractor records matched layer/token metadata and paired tensors at aligned positions. Each sample contains:
\begin{itemize}[leftmargin=1.4em]
    \item prompt identifier;
    \item token position;
    \item layer index;
    \item residual-stream state in both regimes;
    \item optional KV cache slices;
    \item optional sparse logits snapshots.
\end{itemize}

The current paper focuses on residual uplift first. KV uplift remains a secondary object because direct cache prediction is more brittle and was not yet shown to improve the central behavioral metrics.

\subsection{Model families and controls}
The completed experiments cover:
\begin{itemize}[leftmargin=1.4em]
    \item \method{identity}: no correction;
    \item \method{mean-delta}, global linear, and per-layer linear baselines;
    \item \method{lowrank} and \method{mlp} residual translators;
    \item same-site exact-substitution reference patches (formerly called ``oracle'' rows in earlier notes).
\end{itemize}

The strictest control is \method{identity}. If identity differs from the proper no-patch path, the evaluator is invalid. That check now passes exactly.

\subsection{Prompt families and split protocol}
The current evidence is built on synthetic but structured prompt families rather than unconstrained open-ended traces. The prompt pools include arithmetic, symbolic, logical, sequential, and constrained-completion templates. This was a deliberate design choice: the first job of the project was to establish whether the handoff premise is even falsifiable under cleanly paired conditions.

The completed tables in this draft therefore combine:
\begin{itemize}[leftmargin=1.4em]
    \item a pilot split used for fast kill-tests and site sweeps;
    \item a smaller held-out split used in the frozen targeted-site study;
    \item an active larger held-out80 rerun used only as ongoing validation, not as a completed result in this manuscript.
\end{itemize}
The important discipline is that architectural changes are not justified until they survive the held-out handoff metric, not just the pilot tables.

\subsection{Evaluation ladder}
The evaluation hierarchy is intentionally brutal:
\begin{enumerate}[leftmargin=1.4em]
    \item low/full separation sanity checks;
    \item extractor alignment checks;
    \item null controls and broad learned baselines;
    \item live layer sweeps;
    \item suffix-span sweeps;
    \item targeted-site training and held-out validation.
\end{enumerate}

This sequence matters because it eliminates false-success paths. The project explicitly rejects the claim ``the translator works'' if only local MSE improves while continuation remains unchanged.

Once the correct intervention site was isolated, the next question was whether the patch needed more recent context than the final prefix token. The completed suffix-span sweep says mostly no: widening from the last token to the last 4 or 8 tokens did not produce a comparable gain to moving the intervention to the right site.

\subsection{What the evidence now supports}
The current evidence justifies the following claims:
\begin{itemize}[leftmargin=1.4em]
    \item low/full Gemma 3n operating regimes are measurably distinct;
    \item the live handoff path is valid because identity equals proper no-patch exactly;
    \item local residual fit is insufficient as a success criterion;
    \item the first robust learned handoff site is \method{layer34,last1};
    \item same-site exact substitution is a reference patch, not a hard behavioral ceiling;
    \item intervention placement is a more important bottleneck than broad translator capacity.
\end{itemize}

\section{Discussion}
\subsection{Why this is not just ``two different models''}
The right framing is neither extreme. The reduced and fuller regimes are not identical, and the published E2B and E4B artifacts are exposed separately. But for the purposes of this project they are still nested operating regimes of one parent family, trained and structured together rather than unrelated standalone models. That is why same-parent uplift is a coherent hypothesis and why the experiments can legitimately ask whether a small corrective transport suffices.

\subsection{Why \method{layer34,last1} changes the project}
Before the targeted-site study, the project had evidence of regime separation and local learnability but not of meaningful behavioral recovery. The \method{layer34,last1} result changes that. It turns the work from ``interesting internal geometry'' into a practical handoff result with held-out support.

The negative result is equally important. \method{layer16,last1} looked plausible in earlier sweeps and failed under stronger validation. That failure made the final result more credible because it showed the project was not simply confirming whichever site looked best in an early pilot.

\subsection{Why the reference patch is not a ceiling}
At some horizons the learned row exceeds the same-site exact-substitution reference. That means raw state fidelity is not identical to optimal rollout behavior. The sensible interpretation is that the reference patch is a control surface, not a mathematical upper bound for continuation match. This paper therefore uses the term \emph{reference patch} consistently instead of \emph{oracle ceiling}.

\subsection{Limitations}
The larger held-out80 rerun is still active. The earlier held-out validation accidentally used only 10 prompts, so the evaluator was hardened to fail fast on prompt-count mismatch and to write incremental artifacts. That change affects run hygiene, not the core scientific question, but the larger held-out result is still the next validation gate.

The other limitation is scope. This paper establishes one strong residual handoff site first. It does not yet claim that the same uplift map works uniformly across intermediate Mix-n-Match regimes, nor that KV uplift is already competitive.

\subsection{Threats to validity}
Three threats matter most at this stage.
\begin{itemize}[leftmargin=1.4em]
    \item \textbf{Prompt diversity.} The completed held-out result is real, but still limited in breadth. The larger held-out80 rerun exists precisely because prompt-count mistakes can otherwise make a result look stronger than it is.
    \item \textbf{Single-site intervention.} RegimeLift has established a first useful site, not a full theory of all intervention surfaces. Sparse multi-layer reference patches remain stronger than many single-site learned interventions.
    \item \textbf{Objective mismatch.} The project already has evidence that excellent local residual fit does not guarantee good continuation recovery. This means future objective changes must be judged behaviorally, not cosmetically.
\end{itemize}

\section{Conclusion}
RegimeLift now has a stable empirical spine. The project no longer rests on the vague claim that ``state translation might work.'' It now supports a sharper statement:
\begin{quote}
Within Gemma 3n's nested E2B/E4B family, low-to-full regime uplift is behaviorally recoverable at a specific learned handoff site, and broad residual fitting alone is not enough to find that site.
\end{quote}

That is the main result of the current paper. The next step is not to widen the architecture search blindly. It is to finish the held-out80 validation at the already-identified site and then decide whether objective refinement, multi-site learning, or KV-level targets are justified.
